v0.1.7
- Self-healing wizard patcher: the wizard's AUTOUPDATE is
  hardcoded 'No', so existing installs never pull the upstream
  wizard updates. We patch the installed wizard's loginit.py on
  disk to add the Gemini AI + Wyzie Connect Services entries.
  Idempotent + re-injects on every Kodi startup. Now users get
  the entries WITHOUT having to manually reinstall the wizard.
- Wyzie timeout resilience: tightened per-request timeout from
  15s to 8s, added one retry on connection-level failures (with
  2s delay), and short-circuit the rest of a multi-language
  search if Wyzie is clearly offline (no response on first call).
  Prevents the user staring at a 45+ second wait for "no subs
  found" when Wyzie's Cloudflare frontend is having a bad day.
- Test connection + empty-result diagnostics now distinguish
  timeout-vs-other-failure clearly and offer a workaround in
  Hebrew ("click an English sub from All_Subs - the AI hook
  will translate it").

v0.1.6
- RTL punctuation fix. Strengthened the prompt with explicit
  examples ("מה שלומך?" right, "?מה שלומך" wrong) AND added a
  defensive post-processor that automatically moves misplaced
  punctuation from the start of a Hebrew line to the end.
  Verified with 5 unit cases.
- Wyzie language-alias fallback. OpenSubtitles (which Wyzie wraps)
  serves Hebrew under three different codes: 'he', 'heb', and the
  legacy 'iw'. The search now tries all three for any Hebrew
  request and unions the results, so titles indexed under the
  legacy code finally surface. For other languages, no behaviour
  change.
- Detailed empty-result diagnostics. Instead of a generic "Wyzie
  לא מצא כתוביות" the user now sees the actual reason: 0 results
  across all three Hebrew codes (= title not in index), HTTP 401/
  403 (= key rejected), 429 (= daily quota), 5xx (= service down),
  or "no response" (= timeout/network).

v0.1.5
- Test connection button for Wyzie (mirrors the one for Gemini).
  Sanity-checks the key against the live Wyzie API and reports a
  clear Hebrew explanation if quota's exhausted, key's rejected,
  or the service is down.
- New "TMDB" group in settings with an info button explaining
  that gender-aware translation needs TMDB API key wired up via
  the wizard's "Connect Services" screen. The button checks the
  current status (TMDB Helper installed + key set) and tells the
  user what to do next.
- This addon now shows up in the wizard's "Connect Services"
  menu as two entries: "Gemini AI" and "Wyzie". Save / restore /
  display key flow matches the other services there.

v0.1.4
- CRITICAL FIX: undo the auto_translate=false state that v0.1.1
  left behind. Without this, the v0.1.3 hook never fires (DarkSubs
  only calls machine_translate_subs when auto_translate=true) and
  users who'd been through the v0.1.1 takeover ended up getting
  plain English subs on every DarkSubs pick -- no AI AND no Google.
  The restore is gated on the takeover marker, so users who had
  manually turned auto_translate off themselves (no marker) are
  untouched.

v0.1.3
- DarkSubs handshake replaced with proper routing: when the user
  has a Gemini key, DarkSubs's machine-translate now calls into
  OUR AI translator instead of Google/Bing/Yandex. No key set ->
  exactly the same Google flow as upstream (zero regression).
- Self-healing: a service.py-side patcher injects the hook into
  DarkSubs's engine.py on every Kodi startup. If a DarkSubs update
  wipes the hook, it comes back on next launch automatically. If
  DarkSubs ever renames the function, the patcher silently skips
  rather than breaking anything.
- Cache buttons (Clear cache / Clear temp) and the API / Wyzie
  action buttons now have explicit <control type="button"
  format="action"> blocks. Kodi 21 v1 settings schema needs that
  for action-type settings to actually render.
